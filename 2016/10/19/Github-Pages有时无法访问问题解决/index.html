Github Pages有时无法访问问题

Blog主这个星期会到学校后，本来想着上上我亲爱的小博客，看看有没有评论员什么的，却突然发现上不了了，首页还是能正常打开，但comments 和其他类似的分页却死活打不开，Blog主我一开始就蒙了，以为是配置文件上出错了，找来找去找不到，后来就干脆执行

直接初始化了，后来我就后悔了，一大堆东西忘了备份，又重新搭建了一次，搞了一整个下午，结果发现还是一个卵样，后来问了度娘，google，也没能找到靠谱的说法，听说还跟南海事件有关，也有国内将github的dns给污染了，额。。反正我是不懂啦

我用的浏览器是Chrome！

问题找到：

再后来我发现问题主要原因都不在这些上面，而是HTTPS协议的问题，下面试我在github上找到的这么一段话：

隔壁是百度翻译的逗比译文，不过就算没有译文也能大概知道，github从2016.6.15起建立的github pages 将强制使用HTTPS协议，换句话说，如果用了HTTP是打不开的！而我遇到的问题就是这个，首页用了https，其他都跳到http了，结果就是首页打开了，其他的都打不开！

解决办法：

知道问题之后，我就很直接地将网址栏上直接添加https，天真的我以为这次能打开了，却发现结果依然没变，MDZZ！一看网址栏，写的还是http，这我就很纳闷了，果然跟国内对Github还是很不友好，后来，在我的一个高中同学帮助下找到了解决方法，具体看http://www.williamlong.info/archives/3918.html
